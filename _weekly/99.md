## 1 vibe-kanban
- 项目名称：vibe-kanban –  AI 编码代理的编排平台
- 官网链接：https://github.com/BloopAI/vibe-kanban
vibe-kanban号称要让你的Claude code/codex等工具提升10倍的效率，现在我们越来越多的再用AI进行开发，但是同时我也在浪费大量的时间，当AI开始执行一项任务之后，我们就是盯着它闪烁的执行过程。
但是因为AI正在修改内容，所以也不太敢同步去做一些修改。
vibe-kanban就希望改变这种情，它不是那种让传统的手动拖拽任务的看板软件，而是一个能真正指挥 AI 干活的中央调度台。
使用它的话，首先本地要有一个支持的编程 agent，基本主流的都支持。然后npx命令安装。然后就可以开始使用它调度你的工作了。
Vibe Kanban 是一个面向 AI 编码代理的编排平台 ，它可以帮助开发者规划、审查并安全地执行 AI 辅助的编码任务。每个任务都在一个独立的 Git 工作树中运行，让您在充分利用 AI 助手功能的同时，也能完全掌控代码库。它还可以配置GitHub 集成和MCP 集成。
整体来说我觉得它的想法非常好，但是目前还是初期，希望后面可以发展的更加完善。

## 2 skills
- 项目名称：skills – Claude skill
- GitHub 链接：https://github.com/anthropics/skills
Claude skill可以说是最近非常火的一个概念，不得不说anthropic这一年确实提出了很多非常热门的标准。
Claude skill 本质上不是一个“新模型”，而是一种「结构化能力模块」的抽象方式。它定义了一组可复用、可调用、可组合的能力单元。
比较常见的一个对比就SKILL和MCP，MCP 连接使 Claude 能够访问各种工具，而技能则教会 Claude 如何有效地使用这些工具。他们不是重叠而是可以组合使用。
本项目的仓库中有50多个可随时定制的Claude skill，涵盖：文档处理（Word、PDF、PowerPoint）、开发工具（Playwright、AWS、Git）、数据分析、商业与营销、沟通、创意媒体、生产力、项目管理和安全。每个类别都有自己的文件夹，其中包含清晰的SKILL.md文件，详细列出了提示词和逻辑。
当然这里面也不能全部都非常实用，有一些也给人有点鸡肋的感觉。但是可以当成是一个sample，如果想实现自己定制的skill，用来作为基础还是很不错的。

## 3 A2UI
- 项目名称：A2UI – agent生成UI
- GitHub 链接：https://github.com/google/A2UI
A2UI 是由 Google 及社区联合维护的开源协议 + 库集合，用来让 AI Agents（智能体）生成富交互的用户界面，并能在不同平台（Web、移动、桌面）上本地渲染这些界面，而不是只靠纯文本交互。
传统的聊天式交互只靠自然语言往返，针对像“预订餐厅”“填写表格”这样的操作，会产生大量“来回问答 + 结果推理”。A2UI 的思路是让智能体输出一组可渲染的界面结构描述（JSON），而前端客户端按照自己的组件库把它渲染成真实的 UI 组件界面，让用户直接操作。 
换句话说，智能体不再只回答“什么时候预订？”，它可以直接给你一套带日期选择器、时间选择器和确认按钮的完整界面，这样交互更直观、更自然。
以前做一些简单智能体的时候，你可能选择streamlit，那现在可以直接让AI通过A2UI直接生成了。
整体而言，A2UI 正在构建一个面向 AI-driven UI 交互的新标准 —— 让未来的 AI 交互更友好。
## 4 Mole
- 项目名称：Mole – macOS 系统清理工具
- GitHub 链接：https://github.com/tw93/Mole
Mole 是一个用 Go + Shell 打造的开源 macOS 系统清理与优化工具，通过终端界面实现“深度清理、卸载应用、分析磁盘、系统优化”等功能，目标是把多个付费工具（CleanMyMac、AppCleaner、DaisyDisk、Sensei、iStat）集成到一个开源命令行体验里。
可以使用brew来快速安装mole，然后执行命令mo就可以开始使用了。启动后显示交互式菜单界面，你可以用方向键选择不同操作。
它的主要功能包括了清理掉占用空间的“隐形垃圾”、彻底删除应用程序、磁盘可视化分析、系统优化和显示实时系统状态。
另外提示一点，如果担心不安全，可以先 --dry-run 看一遍清理计划，避免误删重要文件。

## 5 fresh
- 项目名称：fresh – 终端文本编辑器
- GitHub 链接：https://github.com/sinelaw/fresh
Fresh 是一个终端里的文本编辑器，主打“像 VS Code / Sublime 那样直觉、非模式（non-modal）”的交互：有完整菜单、命令面板、鼠标支持，尽量做到开箱即用，不要求你先背一套 Vim/Emacs 的快捷键。 
它还很强调性能：设计目标是能高效处理多 GB 级大文件或慢速网络流，并保持很低的内存开销，避免传统编辑器在大文件上卡成 PPT。 
它在功能上也不“寒酸”：文件管理、分屏、搜索替换、多光标、Markdown 预览、LSP（跳转定义/补全/诊断/重命名等）、git 相关能力都覆盖。 
扩展方面，插件用 TypeScript 写，并在沙箱化的 Deno 环境里运行——既能用现代 JS 生态，又不至于把编辑器稳定性当盲盒抽奖。
安装也很“照顾懒人”：提供一键安装脚本，另外也支持 brew、npm等多种方式。

## 6 rendercv
- 项目名称：rendercv – 简历生成器
- GitHub 链接：https://github.com/rendercv/rendercv
RenderCV 是一个面向学术与工程人群的 简历生成器：你用 YAML 写简历内容，然后一条命令 rendercv render xxx.yaml 就能生成排版稳定、间距一致、字体排印讲究的 PDF。
你可以把简历当纯文本进 Git 版本管理；并提供多种主题示例与非常细的设计项（页面尺寸、边距、配色、字体、行距、对齐方式等）。  另外还有 JSON Schema，方便你在编辑 YAML 时获得自动补全和内联说明；同时做严格校验：填错字段会明确告诉你哪里错，避免“渲染出来才发现炸版”。
可以通过
```bash
pip install "rendercv[full]"
```
如果你喜欢这种工程化的简历管理方式，那可以去尝试一下。
