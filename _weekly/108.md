## 1. Everything Claude Code
- 项目名称：Everything Claude Code – Claude Code配置优化插件
- GitHub 链接：https://github.com/affaan-m/everything-claude-code
11万star，6千fork——这个项目可以说把Claude Code玩到了极致。Everything Claude Codes是Anthropic 黑客马拉松获胜者的经过实战检验的配置，它不是一个独立工具，而是一套经过10 个月的高强度实战打磨的完整Claude Code 配置合集，涵盖Agent、Skill、Hook、快捷命令、规则、MCPs等一整套Agent Harness工程化方案。
说实话，现在用Claude Code的人不少，但真正能把效率拉满的人不多。这个项目的核心思路就是与其让模型每次从零开始，不如给它预装好"肌肉记忆"——通过精心设计的Hooks、Rules和MCP配置，让Claude Code在编码时自动加载最佳实践，减少token浪费，提升输出质量。
拿到项目如果不知道如何上手，可以看下作者写的指南，一定要看一下，有快速上手、长篇和安全的指南。建议先看shorthand那个，快速上手。
使用的话，可以在 Claude Code 中添加此仓库为插件市场，然后再安装插件。安装完成后，立即获得仓库中定义的所有 commands / agents / skills / hooks，开箱即用
另外很让人感兴趣的是安全模块AgentShield，专门针对Agent场景做了沙箱隔离、输入过滤和权限管控。现在关于AI安全也是越来越被重视了。
整体来说，如果你是Claude Code的重度用户，这个项目是必须关注的。同时这个项目也是理解"Agent Harness工程化"的一个非常好的教材。

## 2. Project N.O.M.A.D.
- 项目名称：Project N.O.M.A.D. – 离线生存计算机
- GitHub 链接：https://github.com/Crosstalk-Solutions/project-nomad
N.O.M.A.D.的全称是Node for Offline Media, Archives, and Data——一台完全离线运行的"生存计算机"。
想象一下，全球互联网突然消失，云端服务全部瘫痪。你打开电脑，还能做什么？
本项目简单说，就把一台电脑变成一个离线知识中心，里面打包了：离线Wikipedia、医学参考资料、可汗学院课程系统、离线地图、基于Ollama的AI聊天助手、CyberChef数据工具、笔记系统等等。所有工具通过Docker容器编排，一个浏览器界面统一管理。
这种项目非常适合那些末日迷们，就是随时准备好应对“一夜之间全球爆发致命僵尸病毒”。当然实际来说一些及场景中也很有价值，比如灾难应急、野外作业、网络受限地区，或者就是单纯想搞一台不依赖互联网的知识终端。
你只需要一台Ubuntu机器，一条安装命令，就能跑起来。当然它对硬件有一定要求，想要跑本地大模型的话至少需要32GB内存和一块像样的显卡。但如果只是用离线百科和课程系统，4GB内存的设备就能跑。
当然僵尸危机我觉得不太现实，但是最近世界的战争也让人感觉到了很不稳定的感觉。

## 3. Learn Claude Code
- 项目名称：Learn Claude Code – 从零手搓ClaudeCode
- GitHub 链接：https://github.com/shareAI-lab/learn-claude-code
如果看本项目的名字，你可能觉得它是一个学习使用 Claude Code 的指南，但实际上它远比这要硬核——它要带你从零开始，手撸一个类似 Claude Code 的 AI 编程智能体。
当然也不要被智能体给吓住了，它其实也没什么神秘的，就像它在网站一开头直接给出的总结，所有 AI 编程 Agent 共享同一个循环：调用模型、执行工具、回传结果。生产级系统会在其上叠加策略、权限和生命周期层。
当然虽然内容是硬核的，但也不是非常难于上手，它把复杂的 Agent 架构拆成了 12 个Session，每个Session反向工程Claude Code的一个Harness机制：工具系统、上下文压缩、子Agent编排、权限管控、任务系统等等。代码从 84 行一路干到 694 行，每次只加一个机制，让你清清楚看到每一步解决了什么问题。
它还提出了一个很棒的观点：这种Harness工程思维不止适用于编程，还适用于农业、酒店运营、医疗研究、教育等任何需要AI处理复杂任务的领域。那其实这一点我是非常认可的，其实将来随着AI编程的进一步完善，后续我们更需要的思路和认知的提升，而不是局限于某些单纯的技术。

## 4. Page Agent
- 项目名称：Page Agent – 阿里出品的网页智能体
- GitHub 链接：https://github.com/alibaba/page-agent
Page Agent是一个"住在网页里的GUI智能体"——不需要浏览器插件、不需要Python、不需要无头浏览器，只要一段JavaScript代码，就能让网页听懂自然语言指令。他是阿里通义出品，1.4万star，TypeScript编写。
它的核心特点是用纯文本方式解析DOM，而不是截图+视觉模型的方式。这意味着不需要多模态大模型，也不需要特殊的浏览器权限，对性能更友好，集成更简单。
使用上非常轻量，一行script标签就能接入，也可以npm安装。支持SaaS AI Copilot集成，几行代码为你的产品加上 AI 助手，无需重写后端。还有像智能表单填写、无障碍访问等场景。还提供了可选的Chrome扩展和MCP Server，用于跨页面任务。
说实话，现在做浏览器自动化的项目非常多，但Page Agent的"纯前端、纯文本DOM"路线确实是一条差异化路径。它的定位是专为客户端网页增强设计，不是服务端自动化工具。对比需要Python后端的方案，Page Agent更适合嵌入到产品中作为内置功能。
项目里给出了测试可以配合的模型，主要还是以qwen自家的居多，当然优势是qwen模型本地运行比较方便。

## 5. OpenMAIC
- 项目名称：OpenMAIC – 清华多智能体互动课堂
- GitHub 链接：https://github.com/THU-MAIC/OpenMAIC
OpenMAIC是清华大学MAIC实验室出品，发布两周就拿了1.2万star，增长势头很猛。它的全称是Open Multi-Agent Interactive Classroom——一键把任何主题或文档变成一个沉浸式的AI互动课堂。
它先根据课程内容自动搭好了完整的课堂结构：有负责串联全程的 AI 老师，有随时补充细节的技术助教，也有像「阿强」、「木木」这样代表不同基础、不同关注点的 AI 同学。
当你输入一个学习主题，系统自动生成PPT课件、测验题、互动模拟实验，然后由AI老师讲课、AI同学和你一起讨论。AI老师还能在白板上画图、写公式、用语音讲解。课后可以导出可编辑的PPTX或互动HTML页面。
技术上基于Next.js + LangGraph构建，支持OpenAI、Anthropic、Google Gemini、DeepSeek、Grok等多家模型。推荐用Gemini 3 Flash，性价比最高。部署也很方便，支持Vercel一键部署和Docker部署。
它还有一个亮点：内置了OpenClaw集成，可以直接在飞书、Slack、Telegram等聊天工具里生成课堂，零本地配置。这个方向我觉得很有想象力——把AI Agent的能力包装成教育产品，体验上确实比单纯聊天对话要好得多。
作为刚发布v0.1.0的新项目，目前功能还在快速迭代中。
