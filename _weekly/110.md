## 1. claw-code + oh-my-codex
- 项目名称：claw-code – GitHub历史增长最快的项目
- 链接：https://github.com/ultraworkers/claw-code
- 项目名称：oh-my-codex (OMX) – Codex CLI的增强层
- 链接：https://github.com/Yeachan-Heo/oh-my-codex

这周的第一个项目我想做一个1+1的模式。
首先要说的是目前github历史上增长速度最快的项目，短短2周时间，已经达到了恐怖的18W+ star，这就是用rust 重写的ClaudeCode。
对的，上周其实我们也聊到过claude code源码泄露事件，当时有一个哥们拿到源码以后立刻用python和rust完全重写了claude code，那这就是这个项目，上周我做视频的那个节点这个项目搜不到了，后来应该是解禁了，项目介绍里也说finally unlocked，所以感兴趣的朋友就不用找我要代码了，自己去学习吧。
因为这个这个项目其实和上周关联很紧密，但是有没有必要介绍太多吧，所以就作为一个简短的1。另外的1是我还想介绍下他在重写过程用到的 oh-my-codex (OmX) 工具，这是一个构建在 OpenAI Codex 之上的工作流层，他的整个重写过程是这样组织的：team mode 并行 review、ralph mode 持久执行循环、architect 级别验证、codex 驱动实现。
用过 OpenAI Codex CLI 的朋友应该知道，Codex 本身是个不错的代码执行引擎，但在工作流管理上比较弱。
所以，用 Codex 的小伙伴可以学习下，如何来重写项目。
安装和使用也很简单：npm install -g oh-my-codex，然后 omx setup 就行。
注意项目的说明，这个是针对codex cli的强化，如果你是用codex app那就不要使用了，可能会有问题。


## 2. Google AI Edge Gallery
- 项目名称：Google AI Edge Gallery – 谷歌的手机AI模型体验馆
- 链接：https://github.com/google-ai-edge/gallery


Google AI Edge Gallery 是 Google 做的一个“本地 AI 应用商店 + 模型跑分 / 演示平台”，上周的节目里我们也介绍了，谷歌刚刚发布了最新的开源模型 Gemma 4 ，它能以离线的形式运行在这个产品上，使用的完全是 iPhone 本身的 GPU 资源，可以第一时间在手机上体验谷歌最新的模型能力。
而且不只是聊天，它还有很多功能模块：比如 Agent Skills 可以给模型接上维基百科做事实核查，Ask Image 可以拍照识别物体和解视觉谜题，Audio Scribe 能实时转录语音，甚至还有一个 Tiny Garden 小游戏，用自然语言来种虚拟花园。
它底层用的是 Google 的 LiteRT 推理引擎，支持从 Hugging Face 下载各种开源模型，还能自己跑 Benchmark 看每个模型在你手机上的实际表现。
说实话，谷歌做这种端侧AI体验一直是比较超前的，之前他们推出的各种 on-device ML 工具链也一直在迭代。这次做成 Gallery 这种集合形态，对普通用户来说体验门槛确实降低了很多——直接从 Google Play 或 App Store 下载就能用，当然了要自己去切换一下到海外区域，国区目前是没有这个app的。
不过目前主要还是体验和演示性质，如果你好奇"大模型跑在手机上到底是什么体验"，这是一个非常好的入门方式。

## 3. MemPalace
- 项目名称：MemPalace – AI记忆系统的天花板
- 链接：https://github.com/milla-jovovich/mempalace


MemPalace 是一个最近火爆的AI 记忆项目，4月5号才创建，短短4天就冲到了近3万star，而且创始人你是不是也很眼熟，没错，就是那个演《生化危机》的米拉·乔沃维奇。
不是姐妹，怎么咱们演科幻就算了，还玩上真实了，难道你是拿到了红皇后的源代码啦？
MemPalace是干什么的呢？它解决的问题是：你和AI聊的所有内容，每次会话结束就没了。6个月的对话，几千万个token，全消失了。MemPalace 的思路是——全部存下来，也不是一股脑堆进去，而是用"记忆宫殿"的架构来组织。
"记忆宫殿"的原理来自古希腊演说家的记忆术，把要记的东西放进想象中的建筑物的不同房间里。MemPalace 把你的对话按项目、人物分类成不同的"翼"，每个翼下面分"厅"，厅下面分"室"，形成一个可导航的记忆地图。
在 LongMemEval 基准测试上，它的 raw mode 拿到了 96.6% R@5，是迄今公开发布的最高分，而且零API调用，完全本地运行。支持 Claude Code、Cursor、Gemini 等主流工具，通过 MCP 协议集成。
不过有意思的是，项目上线48小时内就被社区扒出了几个问题：比如宣称的"无损压缩"其实是损耗性的，AAAK模式在基准测试上比raw mode低了12个百分点；"+"34%宫殿提升"其实是ChromaDB标准功能的效果。创始人也很快发了公开信承认问题并承诺修复。
说实话，虽然上线就被"打假"，但是反而说明这个项目还真有两下子，还有点挺靠谱的感觉。
安装就一行：pip install mempalace

## 4. Hermes Agent
- 项目名称：Hermes Agent – 会自己进化的AI智能体
- 链接：https://github.com/NousResearch/hermes-agent


Hermes Agent 是 Nous Research 做的一个开源AI智能体，它的核心卖点是"会和你一起成长"——不是那种每次对话完就失忆的Agent，而是有内置学习循环的。
简单说就是：它会在任务完成后自动创建技能（skill），以后遇到类似任务直接复用；它有持久记忆系统，通过 SQLite + FTS5 全文搜索跨会话检索历史对话；它还会定期"自省"，主动把重要知识沉淀下来。
其实伴随着这个项目发展最多的讨论就是和openclaw进行对比，有不少人都说比openclaw更好。当然其实openclaw的记忆模式下配合一些自我进化的skill，也是在希望实现持续成长，所以具体的好坏还是留给大家自行评判了。
Hermes Agent的功能是很全面的的，支持多种后端，包括本地、Docker、SSH、Daytona、modal等。特别是 Daytona 和 Modal 提供了 Serverless 持久化，Agent闲置时自动休眠，有消息时唤醒。
模型方面也是都支持， OpenRouter 上200多个模型、OpenAI、Anthropic、Kimi、MiniMax，还有私有模型。一条命令切换：hermes model。
消息平台支持也很全：Telegram、Discord、Slack、WhatsApp、Signal，还有CLI界面。支持定时任务，用自然语言设置就行。
安装方面也是提供了一键安装命令，而且非常有意思的是它还提供了从 OpenClaw 迁移的工具——hermes claw migrate，能自动导入你的记忆、技能、配置和API密钥。看得出来是想直接抢OpenClaw的用户。

## 5. OpenScreen
- 项目名称：OpenScreen – 免费的录屏神器
- 链接：https://github.com/siddharthvaddem/openscreen


OpenScreen是一款开源免费的录屏软件，绝大多数朋友我相信都用过录屏，这是一个比较刚需的品类。
OpenScreen用 Electron + PixiJS 构建，支持录制指定窗口或全屏，自动或手动添加缩放动画，可以录麦克风和系统音频，支持裁剪、变速、标注（文字、箭头、图片），还能自定义背景（壁纸、纯色、渐变）。导出支持不同比例和分辨率。
OpenScreen也给自己定了一个非常直观的对标，Screen Studio。OpenScreen就是一个更简单的版本，覆盖大部分人的核心需求。对于做产品Demo、教程录制的场景来说，完全够用了。
而且它是 MIT 协议，个人和商业用途都免费。这在录屏工具里确实少见——大部分同类工具要么收费，要么带水印。
目前支持 macOS、Windows 和 Linux。
macOS 上可能需要处理一下 Gatekeeper 的拦截，给终端 Full Disk Access 然后执行 xattr 命令就行。
这里也顺便吐槽一句windows，我是用Mac自带的录屏，之前需要在windows上段内容，折腾了半天，自带的那个真是超级难用。
