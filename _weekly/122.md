## 1. ai-job-search

- 项目名称：ai-job-search - 基于 Claude Code 的 AI 求职工作流
- 链接：https://github.com/MadsLorentzen/ai-job-search

第一个项目 ai-job-search，是一个基于 Claude Code 的 AI 求职框架，最近Star增长超级快，看来全球的人都愁着找工作的问题。它不是单纯优化简历的提示词，而是把找工作这件事拆成了一套比较完整的工作流。

你可以首先把项目克隆到本地，使用ClaudeCode进入。
首先需要执行`/setup`命令，然后基于简历、经历、技能、目标岗位这些资料，建立你的个人档案。这里感觉它应该默认是基于linkedin的简历模板，如果你简历内容不全，也会做一些提问来补全信息。这里有一些我的信息，就模糊一下了。
后面可以用 `/scrape` ，会基于前面的个人档案去搜索岗位，我看到中国的话默认是支持linkedin，可以拿到一个职位的列表。
然后用 `/rank` 批量给岗位打分做初筛，最后可以用 `/apply <url>` 针对某个职位生成定制版 CV 和 Cover Letter，甚至还能用 `/interview` 做面试准备。




我觉得这个项目比较值得借鉴的，是它把个人求职资料变成了一个长期维护的“职业上下文”。在源码里可以看到，它会把信息放到.agent文件夹里，来维护一个自己的记忆体系。还有各种职位的搜索也是用独立的skill来管理，还挺不错的。
但是默认的岗位搜索工具主要面向丹麦市场，比如 Jobindex，但它的核心求职流程是语言和国家无关的。你可以用/add-portal 去添加搜索源，项目会尝试去自动创建一个skill，但是像BOSS直聘、猎聘、智联等中国主流招聘网站，通常风控和反自动化限制很严，所以都不行。

另外之前还有网友投稿他做的简历相关项目，录阶：[Lujie CareerKit](https://github.com/Chozzc/Lujie-Careerkit)。它也是希望做一个完整求职工作台，整体我觉得做的非常不错，值得大家学习支持简历编辑、JD 匹配、投递跟踪、模拟面试和复盘，数据也可以本地保存，只是没有自动搜索职位，需要手动录入。

## 2. PhotoGIMP

- 项目名称：PhotoGIMP - 把 GIMP 改造成 Photoshop 风格的补丁
- 链接：https://github.com/Diolinux/PhotoGIMP

第二个项目 PhotoGIMP，是一个很实用的小工具。它做的事情也非常直接：把开源图片编辑软件 GIMP 的界面、快捷键和默认布局，调整成更接近 Adobe Photoshop 的样子。

如果你没用过 GIMP，可以先把它理解成一个免费的开源 Photoshop 替代品。功能上它能做图片修图、合成、平面设计这些事情，但是很多从 Photoshop 迁移过来的用户，最不适应的就是界面和快捷键。工具位置不一样、面板逻辑不一样、快捷键不一样，刚打开的时候会有点劝退。

PhotoGIMP 解决的就是这个迁移成本。它会重排工具布局，让工具位置更接近 Photoshop；把常用快捷键改成 Photoshop 风格；调整窗口、面板和画布空间；还会加一个自己的启动图和应用图标。简单来说，它不是重新做一个图片编辑器，而是给 GIMP 换一套更适合 Photoshop 用户的配置。

目前新版 PhotoGIMP 是给 GIMP 3.0 以及更新版本用的，支持 Linux、Windows 和 macOS。安装方式本质上就是覆盖 GIMP 的配置文件，所以官方也特别提醒，要先打开一次 GIMP，让它生成配置目录，然后再安装 PhotoGIMP；如果你已有自己的 GIMP 配置，最好先备份。

本项目并不是提升功能，而是解决了一个很现实的问题：开源软件很多时候不是能力不够，而是默认体验和用户习惯不匹配。对设计师来说，肌肉记忆很重要，快捷键和工具位置变了，效率会明显下降。PhotoGIMP 就是用很轻的方式，把迁移门槛降下来。



## 3. caveman

- 项目名称：caveman - 让 AI Agent 少说废话的 skill/plugin
- 链接：https://github.com/JuliusBrussee/caveman

第三个项目 caveman，看名字就很有梗。它的目标是让你的 AI 编程 Agent 说话更短。

你是不是也经常刷到某包的一个梗？就是我用最简单、最直接、最不绕弯子的方式来告诉你什么什么的答案？

现在很多 AI 编程工具都有一个共同问题：干活可以，但太爱解释。明明只是改一行代码，它能给你先来三段背景，再说“我很乐意帮你”，最后再总结一遍。这些话其实会浪费时间，也会占用输出 token。


caveman 的做法很简单：它作为一个 skill 或 plugin，安装到 Claude Code、Codex、Gemini、Cursor、Windsurf、Cline、Copilot 等 Agent 里，要求 Agent 保留技术内容、代码、命令和错误信息，但把表达压缩成更短的风格。项目自己的说法是，同样答案平均减少大约 65% 的输出 token。

它提供不同压缩等级，比如 `lite`、`full`、`ultra`，甚至还有一个文言模式。还有一些配套命令，比如 `/caveman-commit` 用来写更短的 Conventional Commit，`/caveman-review` 用来输出一行式 PR 评论，`/caveman-compress` 可以压缩记忆文件，比如把 `agents.md` 这类长期上下文改得更短。

之前我们也介绍过一些啊优化和减少token的项目，但是caveman的思路和之前的那些不太一样。caveman 主要压缩的是输出，不会减少模型的思考 token，也不会让输入上下文变小。但是也要注意如果你的任务本来就很短，加一个 skill 反而可能不划算。

记得好像在GPT-4o的时候，曾经有过一段时间把输出做得很精简，但是后来好像又把这个事忘了。我们有时候其实很希望 AI 回答干净、直接、可执行的回答。尤其在编程 Agent 里，我就是想知道“哪里错了、怎么改、命令是什么、结果是什么”。少说废话，本身就是一种工程能力。


## 4. Meetily

- 项目名称：Meetily - 本地 AI 会议助手
- 链接：https://github.com/Zackriya-Solutions/meetily

第四个项目 Meetily，是一个隐私优先的 AI 会议助手。它可以在本地捕获会议音频、实时转录，并生成会议总结。

现在 AI 会议助手很多，比如自动进 Zoom、Teams，帮你录音、转写、总结。但是呢，一方面，有一些会议内容很敏感，它可能是一些公司的机密信息，它不让你啊去传到这种第三方的服务商那里。另外呢，就是现在的这种会议助手呢，其实很多也都是限量的，你可能不买会员呢，就用不了多少。

Meetily 的定位就是把这些处理尽量放回本地。它支持本机转录，模型可以用 Whisper ；总结可以用 Ollama 这类本地模型，也可以接自己的在线大模型。录音、转录文本和会议数据都存放在本机，不需要默认上传到云端。

功能上，它支持实时转录、AI 总结、导入已有音频重新转写、麦克风和系统音频同时捕获，还支持多平台，macOS、Windows、Linux 都能用。技术架构上是 Tauri 桌面应用，Rust 后端加 Next.js 前端，这种组合现在在本地 AI 工具里也挺常见。
其实从整个实现上来说呢，它并没有什么特别创新的点。但是，它说实话，很多人在去构思AI创业思路的时候都不选择这个方向，因为它不够新颖。但是它的火热，也让我看到了这种有实际应用价值的东西，还是受大家欢迎。

不过本地方案也有代价。转录质量、速度和模型大小，会受到你机器性能影响；如果想要更强的团队协作、高级导出、自动识别发言人这些功能，项目也有 PRO 版本。

## 5. OfficeCLI

- 项目名称：OfficeCLI - 给 AI Agent 用的 Office 文档命令行工具
- 链接：https://github.com/iOfficeAI/OfficeCLI

最后一个项目 OfficeCLI，它的定位是让 AI Agent 可以直接创建、读取、修改 Word、Excel 和 PowerPoint 文件。

OfficeCLI 的重点不是给人写脚本，而是给 AI Agent 用。它是一个单二进制工具，不需要一堆运行时依赖，命令行就能创建、修改、验证 Office 文件。

它有一个最关键的能力，是内置了渲染和预览。OfficeCLI 可以把 docx、xlsx、pptx 渲染成 HTML 或 PNG，也可以用 `watch` 开一个本地预览，让 Agent 形成“生成 - 看效果 - 再修正”的闭环。

功能上，它支持 Word、Excel、PowerPoint 的读取、创建、修改；可以输出结构化 JSON；可以用路径定位元素，比如第几页、第几个 shape、第几行单元格；还支持模板 merge、批量 batch、resident mode、MCP server、Excel 公式计算和 pivot table。这些能力对 Agent 来说很重要，因为它不需要猜属性名，也不需要硬解析一堆 XML。

我觉得Office CLI的功能是非常实用的，而且它也是一个我们很需要的方向。因为在真正的办公场景当中啊，并不只有Markdown，确实更多的时候要接触到Word、Excel、PPT这一些常见的Office的格式。 当然，Office 文档格式非常复杂，想做到完全兼容并不容易。越复杂的模板、动画、图表和企业格式，越需要实际测试。所以能预览这一点就比较不错。
